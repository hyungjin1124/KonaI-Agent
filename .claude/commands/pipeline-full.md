# /pipeline:full

전체 파이프라인을 실행합니다 (모든 화면 빌드, 캡처, 명세서 생성).

## 워크플로우
1. 프로젝트 상태 검증
2. 모든 화면 빌드 검증
3. 전체 스크린샷 캡처
4. 전체 명세서 생성
5. 접근성 검증
6. 결과 리포트 생성

## 실행 단계

### 1. 사전 검증
- Git 상태 확인 (uncommitted changes 경고)
- `screen-states.json` 유효성 검사
- 필요한 디렉터리 존재 여부 확인

### 2. 빌드 검증
```bash
npm run build
```
- TypeScript 타입 검사
- 빌드 에러 확인
- 번들 크기 체크

### 3. 개발 서버 시작
```bash
npm run dev
```
- 포트 3000에서 실행
- Health check 대기

### 4. 전체 화면 캡처
- `config/screen-states.json`의 모든 화면 순회
- 각 화면의 시나리오별 캡처
  - 초기 상태
  - 입력 상태
  - 결과 상태
  - 에러 상태 (있는 경우)

### 5. 명세서 생성
- 모든 화면에 대해 MD + DOCX 생성
- 통합 명세서 생성 (전체 화면 목록 포함)

### 6. 접근성 검증
```bash
npm run validate:a11y
```
- Axe-core를 사용한 자동 검증
- WCAG 2.1 AA 수준 검사
- 위반 사항 리포트 생성

### 7. 결과 리포트
```
📊 KonaI-Agent 파이프라인 실행 결과

🔍 검증 단계
✅ Git 상태: Clean
✅ 빌드: 성공 (12개 화면)
⚠️  접근성: 3개 경고

📸 캡처 단계
✅ 총 화면 수: 12
✅ 총 스크린샷: 48
⏱️  소요 시간: 3분 24초

📄 명세서 생성
✅ Markdown: 12개
✅ DOCX: 12개
✅ 통합 명세서: 1개

🎯 결과
✅ 전체 성공
📁 출력 디렉터리: outputs/
📊 상세 리포트: outputs/pipeline-report.json
```

## 사용 예시

```bash
User: /pipeline:full

# 또는 CLI에서
npm run pipeline:full
```

## 옵션
- `--skip-build`: 빌드 단계 스킵
- `--skip-a11y`: 접근성 검증 스킵
- `--screens SCR-001,SCR-002`: 특정 화면만 처리
- `--parallel`: 병렬 처리 (더 빠르지만 리소스 많이 사용)

## 예상 소요 시간
- 화면 12개 기준: 약 3-5분
- 화면 50개 기준: 약 15-20분
