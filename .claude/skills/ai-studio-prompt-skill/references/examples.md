# AI Studio Build 프롬프트 예시 모음

다양한 요구사항에 대한 프롬프트 변환 예시.

## 예시 1: CRUD 화면

### 요구사항
```
거래처 관리 화면이 필요합니다.
거래처 목록을 보고, 새로 등록하고, 수정/삭제할 수 있어야 해요.
거래처명, 사업자번호, 담당자, 연락처, 주소 정보가 필요합니다.
```

### 생성된 프롬프트
```markdown
## 화면 목적
거래처 정보를 통합 관리하는 마스터 데이터 관리 화면

## 주요 기능
- 등록된 거래처 목록 조회 및 검색
- 신규 거래처 등록
- 기존 거래처 정보 수정
- 거래처 삭제 (확인 절차 포함)

## 데이터 요소
거래처 정보:
- 거래처코드: C001, C002, ...
- 거래처명: (주)한국전자, 삼성물산, ...
- 사업자번호: 123-45-67890
- 대표자명: 홍길동
- 담당자: 김영희 (010-1234-5678)
- 주소: 서울시 강남구 테헤란로 123
- 등록일: 2024-01-15
- 상태: 활성/비활성

예시 데이터 5-10건 포함

## 사용자 상호작용
- 거래처명/사업자번호로 검색
- 신규 등록 버튼 → 입력 폼
- 행 클릭 → 상세/수정 모드
- 삭제 시 확인 대화상자

## 스타일 방향
- 한글 인터페이스
- 전문적인 ERP 스타일
- 목록과 입력 폼의 명확한 구분
```

## 예시 2: 입력 폼

### 요구사항
```
전표 등록 화면 만들어주세요.
전표번호는 자동생성되고, 전표일자, 적요, 차변/대변 계정과목과 금액을 입력합니다.
분개 라인은 여러 줄 추가할 수 있어야 합니다.
```

### 생성된 프롬프트
```markdown
## 화면 목적
회계 전표를 작성하고 등록하는 분개 입력 화면

## 주요 기능
- 신규 전표 작성
- 분개 라인 동적 추가/삭제
- 차변/대변 합계 자동 계산
- 잔액 검증 (차변 = 대변)

## 데이터 요소
전표 헤더:
- 전표번호: 2024-00001 (자동생성, 읽기전용)
- 전표일자: 2024-03-15 (날짜 선택)
- 적요: "3월 급여 지급"

분개 라인 (다중):
| 구분 | 계정과목 | 적요 | 차변 | 대변 |
|------|----------|------|------|------|
| 차변 | 급여 | 3월분 | 50,000,000 | - |
| 대변 | 보통예금 | 급여이체 | - | 45,000,000 |
| 대변 | 예수금 | 원천세 | - | 5,000,000 |

합계: 차변 50,000,000 / 대변 50,000,000 (일치 ✓)

## 사용자 상호작용
- 분개 라인 추가 (+) / 삭제 (-) 버튼
- 계정과목 검색 선택 (드롭다운 또는 모달)
- 저장 전 잔액 검증
- 임시저장 / 확정 구분

## 스타일 방향
- 한글 인터페이스
- 회계 시스템 전문 느낌
- 숫자 입력 시 천단위 콤마 자동
- 에러 상태 명확히 표시 (잔액 불일치 시)
```

## 예시 3: 리포트/조회

### 요구사항
```
월별 매출 분석 리포트입니다.
기간 선택하면 매출 데이터를 테이블로 보여주고,
Excel 다운로드, PDF 출력 기능이 필요합니다.
```

### 생성된 프롬프트
```markdown
## 화면 목적
지정 기간의 매출 실적을 분석하고 리포트로 출력하는 조회 화면

## 주요 기능
- 조회 기간 설정 (시작일 ~ 종료일)
- 매출 데이터 테이블 조회
- 다양한 형식으로 내보내기

## 데이터 요소
조회 조건:
- 시작일: 2024-01-01
- 종료일: 2024-03-31
- 조회 단위: 월별/주별/일별

조회 결과:
| 기간 | 매출액 | 전기대비 | 목표대비 | 달성률 |
|------|--------|----------|----------|--------|
| 2024-01 | 1,250,000,000 | +5.2% | 1,200,000,000 | 104.2% |
| 2024-02 | 1,180,000,000 | -2.1% | 1,150,000,000 | 102.6% |
| 2024-03 | 1,320,000,000 | +8.5% | 1,300,000,000 | 101.5% |
| **합계** | **3,750,000,000** | +3.8% | 3,650,000,000 | 102.7% |

## 사용자 상호작용
- 조회 버튼으로 데이터 로드
- 테이블 컬럼 정렬
- Excel 다운로드 버튼
- PDF 출력 버튼
- 인쇄 미리보기

## 스타일 방향
- 한글 인터페이스
- 리포트/보고서 느낌
- 인쇄 친화적 레이아웃
- 숫자 정렬 (우측), 증감률 색상 구분
```

## 예시 4: 모니터링/실시간

### 요구사항
```
생산 현황 모니터링 화면이요.
각 라인별 가동 상태, 생산량, 불량률을 실시간으로 보여주세요.
이상 발생하면 알람 표시도 되어야 합니다.
```

### 생성된 프롬프트
```markdown
## 화면 목적
생산 라인의 실시간 가동 현황을 모니터링하는 대시보드

## 주요 기능
- 전체 라인 상태 한눈에 파악
- 라인별 세부 지표 확인
- 이상 상황 실시간 알림
- 자동 새로고침 (30초 간격)

## 데이터 요소
전체 현황:
- 가동 라인: 8/10
- 금일 생산량: 12,450개 (목표: 15,000개)
- 평균 불량률: 1.2%
- 이상 알람: 2건

라인별 상태:
| 라인 | 상태 | 생산량 | 목표 | 달성률 | 불량률 |
|------|------|--------|------|--------|--------|
| A-1 | 🟢 가동 | 1,580 | 1,500 | 105% | 0.8% |
| A-2 | 🟢 가동 | 1,420 | 1,500 | 95% | 1.1% |
| A-3 | 🔴 정지 | 890 | 1,500 | 59% | 3.2% ⚠️ |
| B-1 | 🟡 점검 | - | - | - | - |
| ... |

알람 목록:
- [경고] A-3 라인 불량률 기준 초과 (3.2% > 2.0%)
- [정보] B-1 라인 정기 점검 중 (예상 완료: 14:30)

## 사용자 상호작용
- 라인 카드 클릭 → 상세 팝업
- 알람 확인/해제
- 새로고침 간격 설정
- 전체화면 모드

## 스타일 방향
- 한글 인터페이스
- 제조 현장 모니터 느낌 (어두운 배경도 가능)
- 상태별 색상 코드 명확히
- 숫자 크게, 한눈에 파악
- 실시간 느낌 (업데이트 시간 표시)
```

## 예시 5: 설정/마스터

### 요구사항
```
시스템 설정 화면 필요해요.
회사 정보, 회계 기준, 알림 설정 등을 탭으로 구분해서요.
```

### 생성된 프롬프트
```markdown
## 화면 목적
시스템 전반의 설정을 관리하는 관리자용 설정 화면

## 주요 기능
- 카테고리별 설정 항목 관리
- 설정 변경 및 저장
- 기본값 초기화

## 데이터 요소
회사 정보:
- 회사명: (주)코나체인
- 사업자번호: 123-45-67890
- 대표자: 홍길동
- 주소: 서울시 강남구...
- 로고 이미지 업로드

회계 설정:
- 회계연도 시작월: 1월 / 4월
- 부가세 신고 주기: 분기 / 반기
- 원가 계산 방식: 총평균법 / 선입선출법
- 감가상각 방법: 정액법 / 정률법

알림 설정:
- 이메일 알림: ON/OFF
- 알림 수신 이메일: admin@company.com
- 일일 리포트 발송: ON/OFF (발송 시간: 09:00)
- 이상 알림 임계값: 불량률 2% 이상

## 사용자 상호작용
- 탭 전환으로 카테고리 이동
- 각 항목 수정 후 저장
- 변경사항 있을 시 저장 버튼 활성화
- 저장하지 않고 이동 시 확인 대화상자

## 스타일 방향
- 한글 인터페이스
- 관리자 설정 페이지 느낌
- 깔끔한 폼 레이아웃
- 섹션별 명확한 구분
```

## 복합 화면 예시

### 요구사항
```
주문 관리 화면인데요,
좌측에 주문 목록, 우측에 선택한 주문 상세가 나오고,
상세에서 바로 상태 변경도 할 수 있으면 좋겠어요.
```

### 생성된 프롬프트
```markdown
## 화면 목적
주문 접수부터 완료까지 전체 프로세스를 관리하는 통합 주문 관리 화면

## 주요 기능
- 주문 목록 조회 및 필터링
- 주문 상세 정보 확인
- 주문 상태 변경 처리
- 마스터-디테일 동시 표시

## 데이터 요소
주문 목록:
| 주문번호 | 주문일시 | 고객명 | 금액 | 상태 |
|----------|----------|--------|------|------|
| ORD-2024-001 | 03-15 09:30 | 김철수 | 150,000 | 🟡 접수 |
| ORD-2024-002 | 03-15 10:15 | 이영희 | 280,000 | 🔵 배송중 |
| ORD-2024-003 | 03-14 14:20 | 박민수 | 95,000 | 🟢 완료 |

선택된 주문 상세 (ORD-2024-001):
- 주문자: 김철수 (010-1234-5678)
- 배송지: 서울시 강남구 역삼동 123-45
- 요청사항: "부재시 경비실에 맡겨주세요"

주문 상품:
| 상품명 | 수량 | 단가 | 금액 |
|--------|------|------|------|
| A상품 | 2 | 50,000 | 100,000 |
| B상품 | 1 | 50,000 | 50,000 |
| **합계** | | | **150,000** |

상태 변경: 접수 → 확인 → 배송준비 → 배송중 → 완료

## 사용자 상호작용
- 목록에서 주문 선택 → 우측 상세 표시
- 상태 필터 (전체/접수/배송중/완료)
- 상태 변경 버튼 (단계별)
- 메모 추가

## 스타일 방향
- 한글 인터페이스
- 분할 레이아웃 (좌측 목록 / 우측 상세)
- 상태별 색상 뱃지
- 주문 처리 업무에 최적화
```
