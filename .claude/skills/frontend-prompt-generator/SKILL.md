# Frontend Prompt Generator

Claude Code에게 프론트엔드 개발을 지시하는 효과적인 프롬프트를 생성한다.

## 프롬프트 생성 워크플로우

1. **요구사항 수집** - 사용자로부터 핵심 요구사항 파악
2. **기술 스택 결정** - 프레임워크, 스타일링, 상태관리 선택
3. **프롬프트 구성** - 구조화된 프롬프트 템플릿에 맞춰 작성
4. **프롬프트 출력** - 복사 가능한 형태로 제공

## Step 1: 요구사항 수집

사용자에게 다음 정보를 질문하거나 제공된 정보에서 추출:

| 항목 | 필수 | 예시 |
|------|------|------|
| 프로젝트 목적 | ✓ | CEO 대시보드, 할일 앱, 랜딩페이지 |
| 주요 기능 | ✓ | KPI 차트, CRUD, 반응형 레이아웃 |
| 기술 스택 | △ | React + TypeScript, Vue 3, Vanilla JS |
| 디자인 참조 | △ | 미니멀, Material Design, 특정 사이트 |
| 데이터 구조 | △ | API 응답 형식, 상태 구조 |

**최소 필수 정보**: 프로젝트 목적 + 주요 기능

## Step 2: 기술 스택 결정

사용자가 명시하지 않은 경우 프로젝트 특성에 따라 추천:

```
간단한 정적 페이지 → Vanilla JS + TailwindCSS
인터랙티브 SPA → React + TypeScript + TailwindCSS  
대규모 앱 → React + TypeScript + Zustand/Redux + TailwindCSS
빠른 프로토타입 → React + shadcn/ui
```

## Step 3: 프롬프트 구성

생성할 프롬프트는 반드시 아래 구조를 따른다:

```markdown
# [프로젝트명] 프론트엔드 개발

## 프로젝트 개요
[1-2문장으로 목적과 대상 사용자 설명]

## 기술 스택
- Framework: [React/Vue/Vanilla JS]
- Language: [TypeScript/JavaScript]
- Styling: [TailwindCSS/CSS Modules/styled-components]
- State: [Zustand/Redux/Context API] (필요시)
- 추가 라이브러리: [차트, 폼 등]

## 주요 기능 요구사항
1. [기능 1]: [상세 설명]
2. [기능 2]: [상세 설명]
3. [기능 3]: [상세 설명]

## UI/UX 요구사항
- 레이아웃: [설명]
- 반응형: [브레이크포인트 또는 "모바일 우선"]
- 디자인 톤: [미니멀/모던/기업용 등]
- 컬러 스킴: [주요 색상 또는 "자유롭게"]

## 컴포넌트 구조 (선택)
src/
├── components/
│   ├── [Component1].tsx
│   └── [Component2].tsx
├── hooks/
└── types/

## 데이터 구조 (선택)
[API 응답 형식 또는 상태 인터페이스]

## 제약사항
- [성능, 접근성, 브라우저 지원 등]

## 참고사항
- [추가 컨텍스트, 예시 사이트 등]
```

## Step 4: 프롬프트 출력

생성된 프롬프트를 마크다운 코드블록으로 감싸서 제공한다. 사용자가 바로 복사하여 Claude Code에 붙여넣을 수 있도록 한다.

---

## 프롬프트 작성 원칙

1. **명확성**: 모호한 표현 대신 구체적인 요구사항 기술
2. **구조화**: 섹션별로 정보 분리하여 파싱 용이하게
3. **적정 범위**: 한 프롬프트에 너무 많은 기능 포함 지양
4. **컨텍스트 제공**: 왜 이 기능이 필요한지 배경 설명
5. **예시 활용**: 복잡한 요구사항은 예시 데이터로 보완

## 예시

**사용자 입력**: "CEO가 사용할 매출 대시보드 만들어줘"

**생성 프롬프트**: `references/example-prompts.md` 참조

## 고급 패턴

복잡한 프로젝트나 특수 요구사항의 경우 `references/advanced-patterns.md` 참조:
- 다단계 개발 프롬프트 분할
- API 연동 명세 작성법
- 디자인 시스템 연동 지시
