# Prompts Templates

이 디렉토리는 AI Studio Build에서 사용할 프롬프트 템플릿을 저장하는 공간입니다.

## 🎯 용도

- 에이전트 시스템 프롬프트
- 작업별 프롬프트 템플릿
- 프롬프트 체이닝 시나리오
- Few-shot 학습 예제

## 📂 파일 구조 권장사항

```
prompts/
├── system/           # 시스템 프롬프트
├── tasks/            # 작업별 프롬프트
├── chains/           # 프롬프트 체인
└── examples/         # Few-shot 예제
```

## 📝 프롬프트 작성 가이드

### 시스템 프롬프트
에이전트의 역할, 성격, 제약사항을 정의합니다.

```markdown
# System Prompt: Technical Writer Agent

당신은 기술 문서 작성 전문가입니다.

## 역할
- 명확하고 정확한 기술 문서 작성
- 복잡한 개념을 쉽게 설명
- 일관된 문서 스타일 유지

## 제약사항
- 사실에 기반한 정보만 제공
- 전문 용어 사용 시 설명 추가
- 코드 예제는 실행 가능해야 함
```

### 작업 프롬프트
특정 작업을 수행하기 위한 구조화된 지시사항입니다.

```markdown
# Task: API 문서 생성

다음 API 엔드포인트에 대한 문서를 생성하세요:

## 입력
- 엔드포인트 URL
- HTTP 메서드
- 파라미터 정의

## 출력 형식
1. 엔드포인트 설명
2. 요청 예제
3. 응답 예제
4. 에러 코드 설명
```

## 🔧 프롬프트 파일 형식

- `.md` - Markdown 형식의 프롬프트
- `.txt` - 순수 텍스트 프롬프트
- `.json` - 구조화된 프롬프트 (메타데이터 포함)

## 💡 베스트 프랙티스

1. **명확한 지시**: 모호하지 않고 구체적으로 작성
2. **컨텍스트 제공**: 필요한 배경 정보 포함
3. **예제 사용**: Few-shot 학습을 위한 좋은 예제 제공
4. **출력 형식 지정**: 원하는 응답 형식을 명확히 정의
5. **제약사항 명시**: 하지 말아야 할 것들을 명시

## 📊 템플릿 변수

프롬프트에서 사용할 수 있는 변수 형식:

```
{{variable_name}}
```

예제:
```markdown
{{task_description}}에 대한 기술 문서를 작성하세요.
사용할 기술 스택: {{tech_stack}}
```

## ✅ 프롬프트 테스트

새로운 프롬프트를 추가할 때:
1. 다양한 입력으로 테스트
2. 예상 출력과 실제 출력 비교
3. 엣지 케이스 확인
4. 성능 및 품질 평가
